## Apresentacao da dinamica

Esta aula foi organizada como uma investigacao guiada em banco de dados. Em vez de resolver apenas comandos isolados, os alunos assumem o papel de investigadores e usam SQL para descobrir o que aconteceu em um caso ficticio dentro de uma empresa.

A proposta e fazer com que o estudante pratique consultas de forma contextualizada, entendendo por que esta consultando e como as informacoes de tabelas diferentes se relacionam.

## Objetivo da atividade

Desenvolver o uso de:

- `SELECT`
- `WHERE`
- `JOIN`
- `ORDER BY`

Ao mesmo tempo, a dinamica busca estimular:

- leitura e interpretacao de dados
- raciocinio logico
- atencao aos detalhes
- organizacao de evidencias

## Como funciona

Os alunos recebem um banco de dados ficticio com informacoes sobre usuarios, dispositivos, arquivos, acessos e downloads. A partir dessas tabelas, devem investigar um acesso suspeito e responder perguntas objetivas com base nas evidencias encontradas.

Cada resposta deve ser acompanhada da consulta SQL utilizada.

## Etapas da aula

1. Executar o script do banco de dados.
2. Explorar as tabelas com consultas simples.
3. Identificar o registro suspeito.
4. Relacionar usuario, dispositivo, IP e arquivo.
5. Preencher o formulario com respostas e consultas.
6. Elaborar a conclusao final da investigacao.

## Orientacoes para os alunos

- Leia cada pergunta com atencao.
- Comece pelas consultas mais simples.
- Use os resultados anteriores como pista para a proxima etapa.
- Evite chutar respostas sem consultar o banco.
- Organize bem o SQL para facilitar a correcao.

## Forma de entrega

A entrega pode ser feita pelo formulario HTML da atividade ou em documento separado, desde que contenha:

- a resposta de cada pergunta
- a consulta SQL correspondente
- a conclusao final da investigacao

## Fechamento pedagogico

Ao final da atividade, o ideal e discutir com a turma quais consultas foram mais eficientes, quais dificuldades apareceram e como os relacionamentos entre tabelas ajudaram a chegar na resposta correta. Assim, o aluno entende que SQL nao e apenas sintaxe, mas uma ferramenta de analise e resolucao de problemas.
