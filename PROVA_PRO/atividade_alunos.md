# Cyber SQL: Investigacao Forense em Banco de Dados

## Missao

A empresa ficticia **DataSecure Systems** sofreu um incidente envolvendo acesso indevido a dados internos. Seu papel e atuar como analista forense digital e descobrir o que aconteceu usando apenas consultas SQL.

Voce pode usar:

- `SELECT`
- `WHERE`
- `JOIN`
- `ORDER BY`
- alias de tabelas

Nao use `GROUP BY` nesta atividade.

## Banco de dados

Execute primeiro o arquivo `cyber_sql_caso01.sql`.

## Perguntas da investigacao

Responda com base nas evidencias encontradas no banco:

1. Qual foi o crime identificado?
2. Em que data e horario ocorreu a acao principal da investigacao?
3. Qual usuario esteve envolvido no incidente?
4. Qual endereco IP foi usado?
5. Qual dispositivo foi utilizado?
6. Qual arquivo foi acessado ou baixado de forma suspeita?
7. Em qual setor trabalha o usuario investigado?
8. Existe algum registro de acesso fora do horario normal? Qual?
9. Existe alguma mensagem interna que ajude a comprovar a investigacao? Qual?
10. Quais dados ligam o usuario, o dispositivo, o IP e o arquivo no mesmo caso?
11. Quais consultas SQL comprovam sua resposta?
12. Qual e a conclusao final da investigacao?

## Regras da entrega

- Entregue as respostas em texto.
- Inclua as consultas SQL usadas.
- Organize a investigacao na ordem em que voce encontrou as evidencias.
- Se usar mais de uma consulta para provar uma resposta, inclua todas.

## Dica

Comece pelas tabelas de logs e downloads. Depois relacione os dados com usuarios, dispositivos, IPs, arquivos e mensagens.
